# [Graph Feature Preprocessor: Real-time Extraction of Subgraph-based   Features from Transaction Graphs](https://arxiv.org/abs/2402.08593)

## Write a very high-quality and detailed summary of the paper that describes the paper in a way that a human can fully understand. The summary should cover the problem, the proposed solution and highlight the main contributions of the paper.

 Here is a detailed summary of the key points from the paper:

Problem:
- Financial transaction data is typically represented in tabular formats which limits the ability to uncover complex insights compared to graph representations. However, graph representations pose challenges for real-time processing required in applications like fraud detection.  
- Gradient boosting models like LightGBM and XGBoost are the fastest ML models for tabular data but cannot take into account graph structure or patterns indicative of financial crimes.
- Graph neural networks (GNNs) can leverage graph structure but have high latency, lower throughput, and cannot operate in a streaming manner.

Proposed Solution:
- The authors propose a Graph Feature Preprocessor (GFP) library to generate additional graph-based features from financial transaction graphs. These features capture suspicious patterns like cycles and scatter-gather structures.
- They use GFP with LightGBM and XGBoost in a graph machine learning pipeline to enable real-time detection of suspicious transactions.
- GFP uses a dynamic in-memory multigraph, parallel subgraph mining algorithms, and incremental vertex statistics computation to enable streaming operation.

Key Contributions:
- GFP software library to extract graph-based features from evolving transaction graphs in real-time. Offered in IBM products and as open source.
- Graph machine learning pipeline combining GFP and gradient boosting models to detect financial crimes in a streaming manner.
- Experiments on synthetic AML and real Ethereum datasets show GFP features lead to 36% higher F1 score compared to GNN baselines.
- Pipeline achieved higher throughput on 32 CPU cores than GNN baselines on V100 GPU, demonstrating practical viability.

In summary, the paper introduces a streaming graph feature extraction library and demonstrates its ability to improve performance of financial crime detection compared to state-of-the-art GNNs in terms of accuracy, throughput and latency.


## Summarize the paper in one sentence.

 This paper presents a software library called Graph Feature Preprocessor that extracts subgraph features from financial transaction networks in real time to detect patterns indicative of financial crimes and enrich transaction data for improved machine learning model accuracy.


## What is the main contribution of this paper?

 According to the paper, the main contributions are:

1) The authors present a graph feature preprocessor (GFP) library for fast feature extraction from dynamically changing transaction graphs. The library leverages an in-memory dynamic multigraph representation and fine-grained parallel subgraph enumeration algorithms to achieve real-time performance.

2) They show that the graph-based features generated by GFP can significantly improve the accuracy (up to 46% higher F1 score) of gradient-boosting machine learning models for detecting suspicious financial transactions.

3) They present a full graph machine learning pipeline using GFP and gradient boosting models that can operate in a streaming manner with low latency, high throughput, and competitive accuracy compared to graph neural network baselines.

In summary, the main contribution is the GFP library and associated graph ML pipeline that enables real-time, accurate detection of suspicious financial transactions by extracting graph-based features from transaction networks. The performance and accuracy improvements are demonstrated on synthetic anti-money laundering and real-world Ethereum datasets.
