## Summarize the paper in one paragraph.

The paper proposes HyKnow, an end-to-end task-oriented dialog system that can handle conversations grounded on both structured knowledge (e.g. databases) and unstructured knowledge (e.g. documents). It formulates the task of modeling dialogs involving either or both kinds of knowledge turns. To address this, HyKnow extends the belief state to manage hybrid knowledge, by adding domain-slot-value triples to track goals and a topic slot to retrieve documents. It uses this extended belief state to perform database query and document retrieval, and generates responses based on their results. HyKnow is optimized end-to-end, which allows joint training of structured and unstructured knowledge-grounded dialog modeling. Experiments on MultiWOZ show HyKnow outperforms pipeline systems and has better knowledge retrieval accuracy. The end-to-end framework also makes it more robust when dialogs involve both knowledge types.


## Summarize the paper in two paragraphs.

Here is a two paragraph summary of the paper:The paper proposes a task-oriented dialog system called HyKnow that can handle dialogs involving both structured and unstructured knowledge. Structured knowledge refers to things like databases and ontologies, while unstructured knowledge refers to passages of text or documents. The key innovation is extending the belief state, which tracks the user's goals, to incorporate topics extracted from unstructured knowledge sources. This allows the system to query databases as well as retrieve relevant documents to generate responses grounded in hybrid knowledge sources. The authors implement HyKnow using an end-to-end sequence-to-sequence model with separate encoders and decoders for various components like belief state tracking, document retrieval, and response generation. Experiments on MultiWOZ show HyKnow outperforms baseline task-oriented dialog systems without hybrid knowledge. It also outperforms pipeline approaches that simply combine existing systems with unstructured knowledge modules. Analysis shows the joint training of structured and unstructured knowledge modeling is crucial for HyKnow's strong performance. The results demonstrate the benefits of incorporating diverse knowledge sources in an end-to-end fashion for task-oriented dialog.


## Summarize the main method used in the paper in one paragraph.

Here is a one paragraph summary of the main method used in the paper:The paper proposes a task-oriented dialog (TOD) system called HyKnow for dialog modeling grounded on both structured and unstructured knowledge. The key method is extending the belief state tracking to handle dialog turns involving either structured knowledge like databases (tracked as domain-slot-value triples) or unstructured knowledge like documents (tracked by adding the domain-entity and topic of the turn). Based on this extended belief state, the model performs corresponding database query or document retrieval, and uses the outputs to generate system responses. The overall framework is implemented as an end-to-end sequence-to-sequence model with multiple encoders and decoders, which allows joint optimization of dialog modeling grounded on hybrid knowledge. Two implementations of extended belief state decoding are considered: using a single decoder or separate decoders for structured triples and unstructured topic. Experiments on a modified MultiWOZ dataset with hybrid knowledge show the benefits of the proposed approach.
